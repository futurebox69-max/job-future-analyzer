## 프로젝트 한 줄 정의
**독립형 미래역량 검사 사이트**: 8문항 / 3분 / 6차원 역량 탐색 → 결과 시각화 → BTS 본앱 연결 CTA

## 왜 만드는가
기획서 v2.0 (BTS-최종통합-사업기획서-v2.docx) 기준:
- **Assessment = 유입 엔진** (3-Layer 중 L1). BTS 본앱(Analyzer)과 역할 분리.
- 90일 로드맵 1~3주차 핵심 산출물.
- 목표 KPI: 검사 시작률 45%+, 완료율 70%+, BTS 클릭률 20%+, 공유율 10%+

## 기존 자산
- `bts-assessment-site.html` (886줄) — 프로토타입이 이미 존재. 라이트 테마, 4스크린 SPA 구조, 8문항 검사 포함.
- BTS 본앱: `https://job-future-analyzer.vercel.app` (Next.js 15 + Claude API)
- 본앱 디자인: 다크 테마 (#0A0A0F), 강조 #6C63FF, Noto Sans KR

## 핵심 요구사항

### 1. 기술 구조
- **단일 HTML 파일** (외부 의존성 최소화, CDN 폰트만 허용)
- **별도 도메인/서브도메인에 배포** (BTS 본앱과 독립)
- API 서버 없음. 모든 검사 로직은 클라이언트 사이드.
- **서버 미저장 원칙**: 사용자 응답 데이터를 서버에 전송하지 않음. 브라우저 메모리에서만 처리.

### 2. 검사 구조: 6차원 × 8문항

| # | 차원 | 영문 키 | 설명 | 측정 방식 |
|---|------|---------|------|-----------|
| 1 | 구조적 사고 | structural_thinking | 복잡한 문제를 분해하고 패턴을 찾는 능력 | 시나리오 선택 |
| 2 | 창의적 재설계 | creative_redesign | 기존 방식을 해체하고 새로운 접근법을 만드는 능력 | 시나리오 선택 |
| 3 | 감성 연결 | emotional_connection | 타인의 감정을 읽고 신뢰를 형성하는 능력 | 행동 패턴 선택 |
| 4 | 적응 민첩성 | adaptive_agility | 변화하는 환경에서 빠르게 전환하는 능력 | 상황 대응 선택 |
| 5 | 윤리적 판단 | ethical_judgment | 기술과 비즈니스의 윤리적 경계를 판단하는 능력 | 딜레마 선택 |
| 6 | 협업 지능 | collaboration_intelligence | 다양한 배경의 사람들과 시너지를 만드는 능력 | 팀 시나리오 선택 |
| 7 | 종합 1 | composite_1 | 복수 차원 교차 측정 | 우선순위 배열 |
| 8 | 종합 2 | composite_2 | 복수 차원 교차 측정 | 이미지/직관 선택 |

- 문항 유형: 텍스트 선택, 이미지 선택, 우선순위 드래그 등 다양하게 혼합 (단조로움 방지)
- 각 선택지는 6차원에 대해 서로 다른 가중치를 가짐 (예: A 선택 시 structural_thinking +3, creative_redesign +1)
- **최종 점수**: 각 차원 0~100 정규화

### 3. 화면 구성 (SPA, 4스크린)

#### Screen 1: 랜딩 (유입)
- 히어로: "AI 시대, 당신의 직업보다 먼저 당신의 역량을 보라"
- 3분 / 8문항 / 무료 강조
- 6차원 카드 미리보기
- CTA: "지금 시작하기" → Screen 2로 전환
- **KPI 목표: 시작률 45%+**

#### Screen 2: 검사 진행
- 상단 고정: 진행 바 + 타이머 (3:00 카운트다운, 경고 아님 참고용)
- 문항별 전체화면 카드 (한 번에 한 문항)
- 선택 후 0.5초 피드백 애니메이션 → 자동 다음 문항
- 마지막 문항 후 → "분석 중..." 로딩 (2초) → Screen 3
- **KPI 목표: 완료율 70%+**

#### Screen 3: 결과
- **종합 점수**: 반원형 게이지 (SVG, 0~100)
- **6차원 레이더 차트** (또는 바 차트)
- **역량 유형명**: 점수 조합에 따른 유형 분류 (예: "미래 설계자", "적응형 혁신가" 등)
- **한 줄 판결**: "당신은 OO형 인재입니다. 구조적 사고가 강점이며, 감성 연결을 키우면 더 강해집니다."
- **강점 Top 2 + 성장 포인트 1** 카드
- **공유 버튼**: 카카오톡 / 링크 복사 / 트위터 (결과 카드 이미지 or 텍스트)
  - 공유 시 OG 메타태그 동적 생성 또는 텍스트 기반 공유
- **CTA 1: "내 직업의 AI 대체율도 분석하기"** → BTS 본앱으로 이동 (URL 토큰 포함)
- **CTA 2 (블러 처리): "결정 리포트 받기 (3,900원)"** → 준비 중 안내 (MVP에서는 이메일 수집만)
- **KPI 목표: BTS 클릭률 20%+, 공유율 10%+**

#### Screen 4 (선택): 이메일 수집
- 결정 리포트 대기 등록
- 이메일만 수집, 최소한의 폼

### 4. BTS 본앱 연결 방식
```
// 역량 점수를 Base64 토큰으로 인코딩해 URL 파라미터로 전달
const scores = { st: 85, cr: 72, ec: 68, aa: 91, ej: 56, ci: 78 };
const token = btoa(JSON.stringify({ s: scores, t: Date.now() }));
const btsUrl = `https://job-future-analyzer.vercel.app?competency=${token}`;
```
- BTS 본앱에서 이 토큰을 디코딩해 역량 점수를 결과 화면에 반영
- **평문 점수가 URL에 노출되지 않도록** Base64 인코딩 필수

### 5. 디자인 스펙
- **테마**: 라이트 테마 (배경 #F8F7FF, 기존 프로토타입 유지)
  - BTS 본앱은 다크 테마이므로 검사 사이트는 라이트로 차별화
- **강조색**: #6C63FF (보라), 보조 #4ECDC4 (민트), #FF6B6B (코랄)
- **폰트**: Noto Sans KR (Google Fonts CDN)
- **모바일 퍼스트**: max-width 640px 기준, 터치 최적화
- **애니메이션**: 문항 전환 시 fade/slide, 결과 게이지 카운트업
- **접근성**: WCAG AA 준수, 버튼 최소 44px

### 6. 면책 조항 (필수 포함)

### 8. 이 MVP에서 하지 않는 것
- ❌ 서버 사이드 로직 / API 서버 / DB
- ❌ 회원가입 / 로그인
- ❌ 결제 시스템 (결정 리포트는 "준비 중" 안내만)
- ❌ 다국어 (한국어 전용)
- ❌ 복잡한 심리측정 알고리즘 (MVP에서는 가중치 합산)
- ❌ localStorage/sessionStorage 사용 (서버 미저장 원칙 + 보안)
