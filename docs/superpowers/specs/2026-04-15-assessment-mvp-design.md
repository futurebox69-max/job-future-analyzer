# BTS 미래역량 검사 사이트 MVP — 설계 문서

## 한 줄 정의

기존 `bts-assessment-site.html` (886줄) 프로토타입에 7가지 개선을 추가하여 독립형 검사 MVP를 완성한다.

## 배경

기획서 v2.0 (BTS-최종통합-사업기획서-v2.docx)의 3-Layer 구조 중 Layer 1(Assessment = 유입 엔진)을 독립 사이트로 구현한다. 90일 로드맵 1~3주차 핵심 산출물.

### KPI 목표

| KPI | 목표 |
|-----|------|
| 검사 시작률 | 45%+ |
| 검사 완료율 | 70%+ |
| BTS 클릭률 | 20%+ |
| 공유율 | 10%+ |

### 접근 방식

기존 파일 직접 개선 (A 방식). 단일 HTML 파일 유지. 886줄 → ~1,500줄 예상.

---

## 1. 결과 화면 재설계

기존 결과 화면(Screen 4)의 챗봇을 제거하고, 전환 퍼널 최적화 순서로 재구성한다.

### 화면 구조 (위→아래)

```
① 종합 점수 반원형 게이지 (SVG, 0~100 카운트업)
   + 유형명 + 이모지 + 한 줄 판결
② 6차원 바 차트 (애니메이션 강화, 순차 등장)
   + 강점 Top 2 하이라이트 + 성장 포인트 1
③ 사고방식 메타분석 (기존 유지)
④ 공유 버튼 [링크복사] [X(트위터)]
⑤ BTS 연결 CTA (Base64 토큰 포함)
⑥ 결정 리포트 블러 미리보기 + 이메일 수집 + 마케팅 동의
⑦ 면책 조항 + 저작권
```

### 설계 근거

- 종합 점수로 임팩트 → 상세 분석으로 신뢰 → 공유로 바이럴 → BTS CTA로 전환 → 유료 티저로 업셀
- 챗봇 제거: 결과 확인 → 행동(공유/BTS 클릭) 사이에 마찰을 만들어 KPI를 깎음

### 반원형 게이지

- 순수 SVG, 외부 라이브러리 없음
- 6차원 평균을 종합 점수로 표시 (0~100)
- 로딩 시 0→최종값 카운트업 애니메이션 (1.5초, ease-out)

## 2. 유형 분류 로직

기존 6개 아키타입 → 10개 유형으로 확장. 점수 조합 패턴을 분석하여 분류.

### 분류 우선순위 (위에서부터 먼저 판정)

```
1. avg >= 75 && min >= 60           → 🌟 올라운더
2. range <= 10                      → 🎯 균형 전략가
3. min <= 40 && max >= 90           → 🌋 잠재력 폭발형
4. top2diff <= 5                    → ⚔️ 이중 무기 보유자
5. topKey === 'structural'          → 🏗️ 미래 설계자
6. topKey === 'creative'            → 🎨 창조적 파괴자
7. topKey === 'emotional'           → 💛 공감 리더
8. topKey === 'adaptive'            → ⚡ 적응형 혁신가
9. topKey === 'ethical'             → ⚖️ 윤리 수호자
10. topKey === 'collab'             → 🤝 시너지 메이커
```

### 설계 원칙

- 특수 유형(패턴 기반) 우선 판정 → 나머지는 최고 차원 기반
- 모든 유형이 긍정적 이름 (어떤 결과든 공유하고 싶게)
- 한 줄 판결: 유형별 고정 템플릿 "당신은 OO형 인재입니다. [강점] + [성장 포인트]"

### 점수 정규화

- 기존: max 기준 상대 % → 변경: 절대 0~100 정규화
- 정규화 = (raw 점수 / 이론적 최대) × 100
- 이론적 최대: 8문항 × 평균 최대 가중치 기반 산출

## 3. BTS 연결 + 공유 + 블러 미리보기

### BTS 연결

```javascript
const token = btoa(JSON.stringify({
  s: { st:85, cr:72, ec:68, aa:91, ej:56, ci:78 },
  type: 'adaptive_innovator',
  t: Date.now()
}));
const btsUrl = `https://job-future-analyzer.vercel.app?competency=${token}`;
```

- CTA 클릭 시 새 탭으로 이동
- 본앱 쪽 토큰 디코딩은 이번 MVP 범위 밖

### 공유 기능

| 버튼 | 동작 |
|------|------|
| 링크 복사 | `navigator.clipboard.writeText()` → 토스트 알림 |
| X(트위터) | `window.open()` 트윗 인텐트 URL |

- 카카오톡: MVP에서는 제외 (앱키 등록/심사가 속도를 늦춤), 추후 추가

공유 텍스트 템플릿:
> "AI 시대 미래역량 검사 결과: 나는 ⚡적응형 혁신가! 종합 점수 82점. 당신의 미래 역량은? → [URL]"

### 결정 리포트 블러 미리보기

```

│  │  ░░ 90일 실행 계획 ░░░░░  │
│  └───────────────────────────┘  │
│                                  │
│  📩 출시되면 알려드릴게요        │
│  [이메일 입력________________]   │
│  ☑ 마케팅 정보 수신에 동의합니다  │
│  [알림 신청하기]                 │
│                                  │
│  ※ 이메일은 출시 알림 목적으로만  │
```

- 블러 카드 내 텍스트: 더미 (보이지 않으므로)
- 이메일 수집: Google Forms 숨김 제출 (hidden iframe POST)
- 마케팅 동의 체크박스 미체크 시 버튼 비활성화

## 4. UX 폴리시 + 면책 조항

### 애니메이션

| 위치 | 현재 | 개선 |
|------|------|------|
| 문항 전환 | 즉시 교체 | fade-out → fade-in (0.3초) |
| 선택 피드백 | 텍스트 토스트 | 선택지 하이라이트 + 체크 + 토스트 |
| 결과 게이지 | 없음 | 0→최종값 카운트업 (1.5초, ease-out) |
| 바 차트 | width 0→100% | 순차 등장 (0.2초 간격 stagger) |
| 공유 버튼 | 없음 | 결과 렌더 후 0.5초 뒤 slide-up |

### 분석 중 로딩 화면

문항 완료 → 결과 사이 2초간 가짜 로딩 표시:

- 가짜 진행 바 (0→100%)
- 텍스트 롤링: "6차원 역량 교차 분석 중" → "유형 패턴 매칭 중" → "맞춤 인사이트 생성 중"
- 즉시 결과 표시보다 신뢰감 향상

### 타이머

- 30초 유지, 참고용 표시만 (경고 톤 완화)
- 시간 초과 시 자동 다음 문항 (기존 동작 유지)

### OG 메타태그

```html
<meta property="og:title" content="AI 시대 미래역량 검사">
<meta property="og:description" content="3분 만에 AI 시대 6가지 핵심 역량을 탐색하세요. 무료.">
<meta property="og:type" content="website">
```

### 면책 조항

결과 화면 최하단에 표시:

> "본 검사는 자기인식 탐색 도구이며, 임상적 진단이나 채용 평가의 목적으로 사용될 수 없습니다. 결과는 참고 자료이며, 전문적 진로 상담을 대체하지 않습니다. 검증된 학술 프레임워크(Frey & Osborne, O*NET, McKinsey, Autor, EU AI Act) 기반 설계."

### 기타

- 연도 업데이트: 2025 → 2026
- 용어 통일: "측정" → "탐색" (기획서 과학적 타당성 섹션 반영)
